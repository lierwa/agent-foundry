# `.planning` 中文说明

这个目录是当前项目的 GSD 工作区，作用是把“项目目标、路线图、阶段计划、当前状态”都落到文件里，而不是只存在聊天记录中。

## 你平时只需要关心这几个文件

- `PROJECT.md`
  项目总说明：这项目是什么、当前重点是什么、哪些不做

- `REQUIREMENTS.md`
  需求清单：当前阶段真正需要完成的能力

- `ROADMAP.md`
  阶段路线图：Phase 1 / 2 / 3 分别做什么

- `STATE.md`
  每天开发状态：现在做到哪、卡在哪、下次从哪开始

- `phases/.../*-PLAN.md`
  真正执行任务时要看的具体计划单

## 推荐的最小日常流程

**每天开工第一件事：**
- 看 `STATE.md`
- 看当前 `PLAN.md`

**每天收工最后一件事：**
- 更新 `STATE.md` 里的：
  - 当前状态
  - 最后活动
  - 阻塞 / 风险
  - 下次第一步

## 当前已知限制

- 这个目录还不是 git 仓库
- 所以 GSD 里依赖 commit / branch checkpoint 的部分，暂时不能完整发挥
- 如果后面你想把 GSD 用顺，建议先给这个项目补上 git
